# TD-003 Plan: Decompose `cli/app`

This file is durable handoff context for `TD-003` from `docs/dev/techdebt/techdebt.md`.

## Locked Goal

`cli/app` must stop acting as one frontend plus server-attachment god package. The target state is explicit subdomains with narrow APIs:

- Startup/session target resolution shared by interactive and headless flows.
- Embedded/remote/daemon server attachment and cleanup ownership.
- Auth/onboarding orchestration.
- Project binding.
- Worktree UX.
- Status collection.
- Runtime client attachment.
- Pure Bubble Tea state/controllers.

UI/render/reducer/controller code must depend on typed services/interfaces, not direct filesystem scans, network/RPC calls, auth mutations, git/process execution, runtime-native types, or server composition packages.
